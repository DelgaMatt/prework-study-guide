Prework Study Guide

HTML

  • An Element is usually represented with angle brackets(< >) and is any obtext that sits on your page. HTML elements are ALWAYS CASE SENSITIVE
  • The '!DOCTYPE html' element declaration tells browser to expect HTML on the webpage
  • The 'html lang="en"' element lets browser know the language is in english
  • The 'header' element is always on top of the page. Usually contains a navigation menu and/or introductory content ('h1'and 'img' elements in our case)
  • The 'head' element contains information about the webpage (contains metadata)
  • The 'body' element represents the visible content shown to the user
  • The 'title' element defines the title for a webpage (important for Search Engine Optimization)
  • The 'style' element defines CSS styles associated with the website
  • The 'base' element defines base URL for a webpage
  • The 'link' element connects external resource to the HTML document
  • The 'meta' element defines metadata (charset, description, keywords, author, and viewport) and information about the page used by the browser
  • The 'script' element embeds client side scripts in the HTML document
  • The 'h1-6' element represents a level of heading a given block represents
  • The 'p' element represents a paragraph or block of text
  • The 'div' element creates a division in the page (not semantic HTML)
  • The 'span' element creates inline division (not semantic HTML)
  • The 'img' element contains info about an image
  • The 'br' element create a line break on webpage
  • The 'a' element is an anchor, creating links to the same or other webpages. 'href=' attributes points to the URL for the link (ex: 'a href="..."')
  • The 'footer' element usually contains information regarding the author, copyright, contact, sitemap and navigation
  • The 'sections' elements define a section in a document that contains thematic content
  • The 'ul' element creates unordered lists (bulleit points)
  • The 'ol' element creates ordered lists
  • The 'li' element lists items

  • An Attribute provides further detail about a particular HTML document
  • The 'src' attribute (in 'header' element) defines the source of an image file
  • The 'alt' attribute (in 'header' element) contains a text string that describes the appearance and functionality of the image (important for screen readers and search engine rankings
  • The 'id' attribute identifies elements and selects them

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • Use '*' to apply a rule to all the elements visible on the page
  • Declarations contain two important components: the CSS property we want to apply and the value of the property (color: blue;)
  • Combine two selectors (header, footer) into a single CSS rule with a comma, allowing the declaration to effect both selectors
  • Use element 'link' to link your CSS file to the HTML file
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with Class Selectory ('card' in our case)

Git

  • git status: checks what branch we are currently on
  • git add [file]: adds a file as it looks now to your next commit (stage)
  • git reset [file]: unstage a file while retaining the changes in working directory
  • git diff: diff of what is changed but not staged
  • git diff --staged: diff of what is staged but not yet committed
  • git commit -m "[descriptive message]": commit your staged content as a new commit snapshot
  • git checkout -b branch-name: creates a new branch and switches to it
  • git branch: lists your branches. a * will appear next to the currently active branch
  • git merge [branch]: merge the specific branch's history into the current one
  • git log: show all commits in the current branch's history
  • git config --global user.name "[firstname lastname]": sets a name identifiable for credit when review version history
  • git config --global user.email "[email]": sets an email address that will be associated with each history marker
  • git config --global color.ui auto: set automatic command line coloring for Git for easy reviewing
  • git log branchB...branchA: show the commits on branch A that are not on branch B
  • git log --follow [file]: show the commits that changed file, even across renames
  • git diff branchB...branchA: show the diff of what is in branchA that is not in branchB
  • git show [SHA]: show any object in Git in human-readable format
  • git remote add [alias] [url]: add a git URL as an alias
  • git fetch [alias]: fetch down all the branches from that Git remote
  • git merge [alias]/[branch]: merge a remote branch into your current branch to bring it up to date
  • git push [alias] [branch] transmit local branch commits to the remote repository branch
  • git pull: fetch and merge any commits from the tracking remote branch
  • git rebase [branch]: apply any commits of current branch ahead of a specific one
  • git reset --hard [commit]: clear staging area. rewrite working tree from specified commit
  • git stash: save modified and staged changes
  • git stash list: list stack-order of stashed file changes
  • git stash pop: write working from top of stash stack
  • git stash drop: discard the changes from the top of the stash stack
  • git rm [file]: delete the file from project and stage the removal for commit
  • git mv [existing-path] [new-path]: change an existing file path and stage the move
  • git log --stat -M: show all commit logs with indication of any paths that moved
